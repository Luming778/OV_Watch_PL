# OV_Watch 智能手表软件架构设计

> MCU: STM32F411CEU6 (Cortex-M4F, 512KB Flash, 128KB RAM)
> OS: FreeRTOS v10.3.1
> UI: LVGL
> Display: ST7789V 240x280 RGB565
> Touch: CST816T

---

## 1. 分层架构总览

```
┌─────────────────────────────────────────────────────┐
│                   Application Layer                  │
│  (WatchFace, Settings, Sport, Weather, Music, ...)   │
├─────────────────────────────────────────────────────┤
│                    GUI Layer (LVGL)                   │
│  (Screens, Widgets, Themes, Animations, Fonts)        │
├─────────────────────────────────────────────────────┤
│                 Service / Middleware Layer             │
│  (SensorFusion, BLE Protocol, DataStorage, PowerMgr)  │
├─────────────────────────────────────────────────────┤
│                   OS Layer (FreeRTOS)                 │
│  (Tasks, Queues, Semaphores, Timers, Events)          │
├─────────────────────────────────────────────────────┤
│              Hardware Abstraction Layer (HAL)          │
│  (HWDataAccess + BSP Drivers + STM32 HAL)             │
└─────────────────────────────────────────────────────┘
```

| 层级 | 职责 | 对应目录 |
|------|------|---------|
| Application | 具体页面逻辑、用户交互 | `User/GUI_App/` |
| GUI (LVGL) | UI框架、控件、动画、刷新 | `Middlewares/Third_Party/lvgl/` |
| Service | 传感器融合、BLE协议、数据存储 | `User/Service/` |
| OS (FreeRTOS) | 任务调度、同步与通信 | `User/Tasks/` |
| HAL | 硬件抽象、BSP驱动、STM32 HAL | `User/Func/` + `Drivers/BSP/` |

## 2. FreeRTOS 任务划分

```
优先级    任务名              周期        栈大小       职责
──────    ──────              ────        ──────       ────
High(5)   Touch_Task          10ms       256 words    触摸采样 + 手势识别
High(4)   GUI_Task            5ms(lv_tick) 512 words  LVGL 渲染 + 刷新 LCD
Norm(3)   Sensor_Task         100ms      256 words    传感器采集 (IMU/HR/温湿/气压)
Norm(2)   BLE_Task            事件驱动    256 words    蓝牙收发 + 数据同步
Norm(1)   AppLogic_Task       200ms      256 words    应用逻辑 (计步/消息/闹钟)
Low(0)    PowerMgr_Task       1s         128 words    电源管理 (低功耗/充电检测/看门狗)
```

## 3. 任务间通信机制

```
                ┌─────────────┐
                │  GUI_Task   │◄──── lv_timer_handler() 驱动
                │  (LVGL主循环) │
                └──────┬──────┘
                       │ 消息队列 (gui_msg_queue)
        ┌──────────────┼──────────────┐
        ▼              ▼              ▼
  ┌──────────┐  ┌───────────┐  ┌───────────┐
  │Touch_Task│  │Sensor_Task│  │AppLogic   │
  │          │  │           │  │  _Task    │
  └────┬─────┘  └─────┬─────┘  └─────┬─────┘
       │              │              │
       │  ┌───────────────────────┐  │
       └─►│  Shared Data (Mutex)  │◄─┘
          │  - touch_data         │
          │  - sensor_data        │
          │  - app_state          │
          └───────────┬──────────┘
                      │
              ┌───────▼───────┐
              │  EEPROM保存   │ (低频写入)
              └───────────────┘
```

| 通信方式 | 用途 | 示例 |
|---------|------|------|
| **消息队列** | 任务间单向数据传递 | Touch → GUI（手势事件）、BLE → AppLogic（通知） |
| **互斥锁 (Mutex)** | 保护共享数据 | `sensor_data` 结构体（多读单写） |
| **二值信号量** | 同步事件通知 | LVGL flush完成回调 → GUI_Task 唤醒 |
| **事件组 (EventGroup)** | 多标志位组合 | BLE连接状态、充电状态等 |

## 4. LVGL 集成方案

### 显示缓冲策略（128KB RAM 约束）

| 方案 | 缓冲大小 | 效果 | 推荐 |
|------|---------|------|------|
| 方案A: 双缓冲 | 240×40×2 = 19,200 字节 | 流畅，无撕裂 | **推荐** |
| 方案B: 全屏缓冲 | 240×280×2 = 134,400 字节 | 最流畅，但RAM不够 | 不可行 |
| 方案C: 部分缓冲 | 240×20×2 = 9,600 字节 | 省RAM，有轻微撕裂 | 备选 |

### flush 回调流程

```
lv_timer_handler()
    │
    ▼
flush_callback(area, color_p)
    │
    ▼
SPI DMA 传输 (DMA2_Stream2)
    │
    ▼
DMA 传输完成中断
    │
    ▼
lv_display_flush_ready() 通知 LVGL
```

### 触摸输入驱动对接

```
CST816T 硬件 (软件I2C, PB4/PA15)
    │
    ▼
Touch_Task (10ms)
    │ i2c_read → 坐标 + 手势
    ▼
lv_indev_data → LVGL 输入设备
    │
    ▼
LVGL 事件分发 → 当前页面回调
```

| 模块 | 预估大小 | 说明 |
|------|---------|------|
| FreeRTOS 堆 (heap_4) | 15,360 字节 | 已配置 |

| 全局变量/BSS | ~5,000 字节 | 驱动层全局变量 |

## 7. 页面管理器设计

智能手表的核心是**页面栈管理**，类似手机的 Activity 栈。

### 页面栈

```
┌─────────────────────────┐
│   Notification Page     │ ← 栈顶 (当前显示)
├─────────────────────────┤
│   Sport Detail Page     │
├─────────────────────────┤
│   Menu Page             │
├─────────────────────────┤
│   WatchFace Page        │ ← 栈底 (常驻)
└─────────────────────────┘
```

### 页面操作

- **上滑/下滑** → push 新页面
- **左滑/右滑** → pop 回上一页
- **按键** → pop 到表盘

### 页面接口定义

```c
typedef struct {
    void (*on_create)(void);        // 页面创建（初始化LVGL控件）
    void (*on_resume)(void);        // 页面恢复到前台
    void (*on_pause)(void);         // 页面切到后台
    void (*on_destroy)(void);       // 页面销毁（释放LVGL控件）
    void (*on_gesture)(gesture_t);  // 手势处理
} page_ops_t;
```

```

              │ lv_label_set_text(hr_label, "72")

```

## 9. 与现有代码的适配要点

| 现有模块 | 适配内容 |
|---------|---------|
| **HWDataAccess** | 保持不变，作为 BSP 和 Service 之间的桥梁 |
| **IIC 软件 I2C** | `iic_bus_t` 结构体支持多总线，直接复用 |
| **LCD 驱动** | 添加 LVGL `flush_callback`，利用已有 SPI DMA |
| **Touch 驱动** | 将 CST816T 手势/坐标读取对接 LVGL `indev` 驱动 |
| **FreeRTOSConfig.h** | 增加堆大小、启用互斥锁和消息队列支持 |
| **TIM1 作为 HAL timebase** | 保持不变，SysTick 留给 FreeRTOS tick |
